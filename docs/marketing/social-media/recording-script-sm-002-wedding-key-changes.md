## Goal

Capture a tight wedding/corporate gig workflow:

1. musician has a quick chart,
2. singer changes the key,
3. chart is duplicated,
4. duplicate is transposed,
5. PDF is ready to send.

Target final edit length: 35-45 seconds.

## Capture Setup

- Record portrait iPad screen captures.
- Use Do Not Disturb if possible.
- Hide private notifications and account information.
- Keep the app full-screen and readable.
- Pause briefly before and after each action so cuts have handles.
- Use only rights-safe generic chart material from `demo-material-packet-002.md`.

## Pre-Capture App State

Recommended setup before recording:

- Project: `Wedding Set - Key Changes`
- Chart: `First Dance - C`
- Optional duplicate-ready chart name: `First Dance - F`

If creating the project on camera feels slow, create it beforehand and start from the Projects list.

## Clip Plan

### Clip 1: Context / Project Proof

Length: 5-7 seconds

Action:

- Open Projects.
- Show `Wedding Set - Key Changes`.
- Show `First Dance - C` and, if already created, `First Dance - F`.

Useful pause:

- Hold on the project/chart list for 1 second.

Edit purpose:

- Establish that this is a real gig set, not a feature menu tour.

### Clip 2: Quick Chart Entry

Length: 12-16 seconds

Action:

- Open `First Dance - C`.
- Write 4-8 bars from the C version:
  - `Cmaj7`
  - `Am7`
  - `Dm7`
  - `G7`
  - optional second line: `Cmaj7 | Em7 A7 | Dm7 G7 | Cmaj7`

Useful pause:

- Hold after the chart is readable.

Edit purpose:

- Show paper-speed clean chart creation.

### Clip 3: Key-Change Note

Length: 5-8 seconds

Action:

- Add a handwritten or text note:
  - `Singer wants F`
  - or `vocal key: F`

Useful pause:

- Hold with the note visible.

Edit purpose:

- Make the problem instantly understandable to non-musicians and musicians.

### Clip 4: Duplicate / Rename

Length: 8-12 seconds

Action:

- Navigate to chart/project actions.
- Duplicate the chart or use Duplicate Variant.
- Rename or show the duplicate as `First Dance - F`.

Useful pause:

- Hold on the duplicate/variant confirmation or the renamed chart row.

Edit purpose:

- Show that the new key version is a reusable chart, not a temporary visual trick.

### Clip 5: Transpose To F

Length: 8-12 seconds

Action:

- Open the F duplicate.
- Use the transpose control.
- Show the chart changed from C-family chords to F-family chords:
  - `Fmaj7`
  - `Dm7`
  - `Gm7`
  - `C7`

Useful pause:

- Hold on the transposed result for 1-2 seconds.

Edit purpose:

- This is the payoff. Keep it very clear.

### Clip 6: Export / Share

Length: 8-10 seconds

Action:

- Tap Export.
- Show PDF preview.
- Open Share if it is clean and readable.

Useful pause:

- Hold on the PDF preview.

Edit purpose:

- Close the loop: chart is ready for the gig.

### Clip 7: Clean Close

Length: 4-6 seconds

Action:

- Show the finished F chart full-page or the iChart logo/end card.

Useful pause:

- Hold steady.

Edit purpose:

- Provide a cover frame, end-card option, or quiet CTA landing.

Wedding gigs move fast. You learn the songs, and then the singer changes the key.

With iChart, you can write a clean chord chart by hand, duplicate it, transpose it, and export a PDF without rewriting the page.

It keeps the speed of paper, but the chart stays editable for the next gig.

Testing on iPad now.

## On-Screen Text

Use one hook, then keep the rest minimal:

- `Singer moved it to F`
- `Write the chart`
- `Duplicate it`
- `Transpose to F`
- `Export the PDF`

Avoid filling the screen with paragraph text. The app action should carry the proof.

## Capture QA

- [ ] The C chart is readable before transposition.
- [ ] The F chart is readable after transposition.
- [ ] The recording shows a duplicate or saved version, not only a temporary transpose.
- [ ] No private account, email, notification, or tester information is visible.
- [ ] The chart material is original/generic.
- [ ] No wording implies automatic chart cleanup or full arrangement generation.
- [ ] Export/PDF proof is visible if possible.
