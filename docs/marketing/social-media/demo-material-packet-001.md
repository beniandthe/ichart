## Purpose

Define the rights-safe demo content for the first recording session.

SM-001 should show a real bandleader/gigging-musician workflow:

> Write one clean chart quickly, save it in a project, make chart versions for Concert/Bb/Eb players, and export/send PDFs.

This is not a messy-paper cleanup demo. It is paper-speed digital chart writing with structured reuse.

## Chart Content Guidance

Keep the chart original and generic.

Good content:
- 8-16 bars.
- Simple funk harmony.
- Clearly readable chord symbols.
- A section label or two.
- A repeat/vamp/solo cue if it feels natural.
- A turnaround or ending that gives the page musical shape.

Avoid:
- Recognizable copyrighted songs.
- Famous funk progressions that invite comparison.
- Real client, band, wedding, school, or venue names.
- Full notation claims or written horn lines.

## Suggested Chart Artifacts

Use only what feels natural in the actual chart:

- Section label: `A`
- Section label: `B`
- Text cue: `Vamp`
- Text cue: `Solo`
- Repeat span or `2x`
- Ending marker, if easy and visually clean
- `D.S.` / `Fine` only if it makes the chart clearer
- A simple form cue or accent note if it is visually reliable in the current build

Goal:
- Make the chart feel like a working band chart, not a blank feature demo.

## Suggested Chord Vocabulary

Use what feels right while recording. These are safe original-demo options, not a required progression:

- `Fm7`
- `Bb7`
- `Eb9`
- `Ab13`
- `C7sus`
- `Db9`
- `G7alt`
- `Cm9`
- `F7`
- `Bb13`

If handwriting recognition is the focus of the shot, favor chord symbols that are reliable and readable on camera.
