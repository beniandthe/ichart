## Production Checklist

### Capture

- [ ] Uses rights-safe demo chart or cleared material.
- [ ] No private account, email, support, or tester information visible.
- [ ] Product UI is readable on a phone screen.
- [ ] Extra framing exists for vertical safe-zone crop.
- [ ] Audio/voiceover source is clean.

### Edit

- [ ] 1080x1920 9:16 master exported.
- [ ] Hook appears in first 1-3 seconds.
- [ ] Product action is visible before explanation becomes abstract.
- [ ] Captions or text overlays are readable.
- [ ] CTA is visible and not near platform UI.
- [ ] Brand color/logo use matches `docs/branding/ichart/brand-system.md`.
- [ ] App Store badge is used only when appropriate and not modified.
- [ ] Apple App Store screenshots, app previews, and custom product page claims match the current shipping app.

### Review

- [ ] No unsupported product claims.
- [ ] No full-notation, playback, melody-entry, collaboration, or cross-platform launch implications.
- [ ] Basic/Pro claims match current policy.
- [ ] Music/chart rights are cleared.
- [ ] Accessibility: captions, contrast, alt text or descriptive caption context.
- [ ] Platform-specific caption/length/aspect requirements checked.

### Publish

- [ ] Platform captions are tailored.
- [ ] YouTube Shorts-specific packaging is not included unless the channel decision changes.
- [ ] Hashtags are relevant and not stuffed.
- [ ] Thumbnail/cover selected.
- [ ] Link, App Store path, or custom product page URL tested.
- [ ] Scheduled or published.
- [ ] Comment monitoring window assigned.
