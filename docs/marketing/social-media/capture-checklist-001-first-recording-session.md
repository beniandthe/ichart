## Purpose

Capture the first reusable product footage for TikTok, Instagram Reels, Instagram Stories, Apple screenshots/app-preview storyboards, and later landing-page assets.

The footage must prove the actual iChart product promise:

> Handwrite clean charts at paper speed, then transpose, duplicate, export, store, and edit them like structured iPad charts.

Do not capture or edit this as "messy paper becomes clean." The story is paper-speed creation with structured digital reuse.

## Product Truth To Show

Verified app-facing surfaces/labels to capture where possible:

- `New Chart`
- `Simple Chord Sheet`
- `Rhythm Section Sheet`, only if useful for the chosen demo
- `Duplicate`
- `Duplicate Variant`
- `Instrument Transposition`
- `Concert`
- `Bb Horn`
- `Eb Horn`
- `Export`
- `Export PDF`
- PDF preview
- `Share`
- `PDF Library`

Use precise language:

- Say "chart versions for Bb/Eb players."
- Say "transpose without rewriting."
- Say "send PDFs to the band."
- Do not say "horn parts" unless the visual clearly means chord-chart versions, not written horn notation.
- Do not imply full notation, melody entry, playback, automatic arrangement, or automatic paper-chart cleanup.

## Capture Setup

Device and framing:
- Record iPad screen in portrait-friendly framing where possible.
- If using landscape iPad capture, leave enough room to crop to 9:16 around the action.
- Keep the chart page, active menu, and final PDF readable on a phone.
- Avoid showing email addresses, account details, support screens, TestFlight metadata, private forum data, or real client/song names.

Audio:
- Capture clean screen footage first.
- Voiceover can be recorded later.
- Use platform-cleared music or no music until publishing.

Brand:
- Let the product surface carry the visual.
- Use the logo only for the close frame or cover frame if it does not distract from the chart.

## SM-001 Capture: Full Band / Bb-Eb Chart Versions

Goal:
- Show a bandleader writing one clean chart quickly, making player-ready chart versions, and exporting PDFs.

Primary hook:
- "Write one chart. Make band-ready versions. Send the PDFs."

Capture sequence:

1. Library start
   - Show `Charts`.
   - Tap `New Chart`.
   - Choose the chart type that best fits the demo.

2. Create the chart
   - Use the rights-safe title `Funk Groove`.
   - Save it in a project named `Funk Groove`.
   - Set an ordinary key and meter.
   - Keep setup quick; this is not a settings tutorial.

3. Write the clean chart
   - Use Apple Pencil input or the closest reliable demo path.
   - Capture chord entry, measure flow, and any roadmap cue that makes it feel like a real band chart.
   - Show the chart becoming usable through direct writing, not through paper cleanup.

4. Make a chart version
   - From the library or project context, capture `Duplicate` or `Duplicate Variant` if the current app path is available.
   - Name versions clearly if useful:
     - `Funk Groove - Concert`
     - `Funk Groove - Bb Horn`
     - `Funk Groove - Eb Horn`
   - If the project-specific path is cleaner, capture `Duplicate Variant` and `Instrument Transposition`.

5. Show transposition
   - Capture `Instrument Transposition`.
   - Show `Concert`, then switch to `Bb Horn` or `Eb Horn`.
   - Make sure the visible chord/key change is readable.
   - If needed, capture this in the editor Page menu where `Instrument (...)` appears.

6. Export
   - Capture `Export` or `Export PDF`.
   - Show PDF preview.
   - Capture `Share`.
   - Optional: show the item in `PDF Library`.

7. Close frame
   - Use a clean chart or PDF preview.
   - Text option: "Write once. Make versions. Send PDFs."
   - CTA option: "Testing on iPad now."

Must-have clips:
- `New Chart` -> chart page.
- Handwritten/Apple Pencil chart entry.
- `Instrument Transposition` with `Concert`, `Bb Horn`, or `Eb Horn`.
- `Export`/PDF preview.
- `Share`.

Nice-to-have clips:
- `Duplicate` or `Duplicate Variant`.
- PDF Library storage.
- Physical Apple Pencil hand shot.

## SM-002 Capture: Wedding Gig Key Changes

Status:
- Capture-ready via `docs/marketing/social-media/demo-material-packet-002.md`.
- Recording script: `docs/marketing/social-media/recording-script-sm-002-wedding-key-changes.md`.

Goal:
- Show a gigging musician making quick clean chord charts when singers/client requests move songs into new keys.

Primary hook:
- "POV: the singer moved three songs and the band still needs charts."

Capture sequence:

1. Context opener
   - Use text overlay later, not private notes.
   - Suggested overlay: "Singer moved it to Eb."
   - Use generic song labels such as `First Dance`, `Cocktail Hour`, `Last Dance`.

2. Create or open chart
   - Tap `New Chart` if starting from scratch.
   - Use a simple chord-chart layout if it best communicates quick gig-prep chart writing.

3. Write the chart
   - Capture quick chord chart entry.
   - Keep the form short and readable.
   - Avoid recognizable copyrighted song forms.

4. Transpose
   - Capture the transposition moment.
   - Prefer showing a clear before/after key or chord-symbol change.
   - Use wording "transpose without rewriting," not "auto-arrange."

5. Export/share or store
   - Capture `Export`.
   - Show PDF preview.
   - Capture `Share`.
   - Optional: show chart remains in `Charts` or PDF appears in `PDF Library`.

6. Story poll add-on
   - Instagram Story prompt: "How often do keys change before the gig?"
   - Options: "Constantly" / "Enough to matter"

Must-have clips:
- `Wedding Set - Key Changes` project context.
- `First Dance - C` quick chart entry.
- Key-change note: `Singer wants Eb`.
- Duplicate or saved `First Dance - Eb` version.
- Transpose/key-change moment.
- PDF preview or share.

Nice-to-have clips:
- Library storage.
- Physical Apple Pencil hand shot.
- Story poll still.

## Editing Notes

TikTok:
- Start with the practical pressure.
- Keep UI cuts faster.
- Use captions that sound like musician speech.
- Avoid overly polished ad pacing.

Instagram Reels:
- Same core action, slightly cleaner cover frame.
- Use stronger product legibility.
- Favor save/share value over pure trend language.

Instagram Stories:
- Use polls and beta-progress updates.
- Stories can be more conversational and less edited.

LinkedIn/Facebook/Threads:
- Do not force these into the first capture if the footage is not ready.
- Reuse the strongest proof after TikTok/Reels show which wording lands.

- [ ] Transposition language matches current app behavior.

- [ ] There is at least one clean close frame for cover/thumbnail.
